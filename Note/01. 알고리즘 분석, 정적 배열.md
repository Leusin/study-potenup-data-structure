## 알고리즘 분석

### 최악 분석

- 특정 알고리즘이 해결해야 하는 모든 입력값 중에서 가장 비효율적인 경우를 상정하여 성능을 측정하는 방법
- 어떤 상황에서도 이보다 더 나빠질 수는 없다는 안전한 상한선(upper bound)을 보장

### O표기법

- 함수에서 가장 빨리 증가하는 항만을 남긴 채 나머지를 다 버리는 표기법

### 계산 복잡도

$O(1)<O(logN)<O(NlogN)<O(n^2)<O(2^n)<O(n!)$

- O(n!) - 집합의 모든 순열 생성

## 배열(array)

### 정적 배열(static array)

- 스택에 할당되는 배열은 컴파일 시점에 크기를 알아야 함
- 힙에 할당되는 배열은 컴파일 시점에 크기를 지정하며, 할당 해제를 해서 메모리 누수가 나지 않게 관리

### 동적 배열(dynamic array)

- 힙에 할당되어 자동으로 크기가 늘어나는 배열
- 추상 자료형
    - at(integer i) : index i에 대해서 데이터 값을 반환
    - set(integer i, object o) : index i의 데이터를 o로
    - insert/pushback/emplace(integer i, object o) : index i에 o를 삽입
    - erase(integer i) : index i에 있는 데이터를 제거
    - size() : 크기 반환
    - empty() : 비었는지
